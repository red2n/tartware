## Check-in Requirements by Market

| Market Segment | ID Required | Payment | Registration Card |
|----------------|-------------|---------|-------------------|
| Transient | Government ID | Credit card auth | Signed |
| Corporate | Company ID acceptable | Direct bill possible | Signed |
| Group | ID + rooming list | Master bill or individual | Batch or individual |
| OTA Prepaid | ID | None (prepaid) | Signed |
| Walk-in | Government ID | Full prepay or deposit | Signed |
| International | Passport | Credit card | Police report (some countries) |

## Check-in Types

| Type | Description | Requirements |
|------|-------------|--------------|
| **Standard** | Traditional desk check-in | Staff assistance |
| **Express** | Pre-registered, key ready | Confirmation only |
| **Kiosk** | Self-service terminal | ID scan, card swipe |
| **Mobile** | App-based check-in | Digital key capable locks |
| **Curbside** | Vehicle check-in | Bell staff assistance |

## Check-out Process

| Step | Action | System Update |
|------|--------|---------------|
| 1 | Review folio charges | Display balance |
| 2 | Verify minibar | Post charges if needed |
| 3 | Handle disputes | Adjustments if warranted |
| 4 | Collect payment | Post payment, close folio |
| 5 | Deactivate keys | Key system integration |
| 6 | Update room status | Set to VD (Vacant Dirty) |
| 7 | Email folio copy | Guest communication |

## Express Check-out Options

| Method | How It Works |
|--------|--------------|
| **Key drop** | Guest leaves key, charges to card on file |
| **Kiosk** | Self-service folio review and payment |
| **Mobile** | App-based checkout, digital key deactivation |
| **Auto-checkout** | System-triggered at scheduled departure |
| **Video checkout** | Video call with front desk |

## Room Move Scenarios

| Scenario | Action | Folio Impact |
|----------|--------|--------------|
| Guest request | Assign new room, update housekeeping | Transfer charges |
| Maintenance issue | Emergency move, consider compensation | Possible adjustment |
| Upgrade (courtesy) | Better room, same rate | Log upgrade reason |
| Upgrade (paid) | Better room, charge difference | Post upgrade charge |
| Downgrade | Guest-requested or oversell | Rate reduction, comp |

## Walk Procedures (Oversell)

When property is overbooked and must relocate a guest:

| Step | Action |
|------|--------|
| 1 | Identify walk candidate (typically no loyalty, low rate) |
| 2 | Secure comparable room at alternate hotel |
| 3 | Contact guest before arrival if possible |
| 4 | Provide transportation to alternate |
| 5 | Cover first night at alternate property |
| 6 | Offer compensation (future stay, points) |
| 7 | Guarantee room for return if multi-night |

## Service Recovery

| Issue Level | Response | Compensation |
|-------------|----------|--------------|
| Minor (slow service) | Apology | None or small gesture |
| Moderate (room issue) | Management apology | Room move + amenity |
| Major (safety concern) | Immediate action | Comp night + future discount |
| Critical (injury, police) | Emergency protocols | Case-by-case, legal involved |

## Communication Scripts

### Check-in Greeting
```
"Good [morning/afternoon/evening], welcome to [Hotel Name].
May I have your name or confirmation number?"
```

### Room Ready
```
"[Guest Name], your room is ready. You'll be in room [#] on
the [floor] floor. Here are your keys - the elevator is [direction]."
```

### Room Not Ready
```
"I see you've arrived early. Your room is being prepared and
should be ready by [time]. May I store your luggage and offer
you a complimentary [coffee/access to lounge]?"
```
